排序算法的分类

内部排序：指将需要排序的所有数据都加载到内存中进行排序。

外部排序：数据量过大，无法全部加载到内存中，需要借助外部存储进行排序。

度量算法执行时间的两种方法

1、事后统计的方法
这种方法可行, 但是有两个问题：一是要想对设计的算法的运行性能进行评测，需要实际运行该程序；二是所得时间的统计量依赖于计算机的硬件、软件等环境因素。这种方式，要在同一台计算机的相同状态下运行，才能比较那个算法速度更快。

2、事前估计的方法
通过分析某个算法的时间复杂度来衡量哪个算法更优。

算法的时间复杂度

时间频度

一个算法中的语句执行的次数，记为T(n)

例如：

int total = 0;
int end = 100;
for(int i=1;i<=end;1++){
    total++;
}

循环执行n次，最后再进行一次i<=n的判断，故 T(n) = n+1

直接计算：total = (1+end)*end/2 ， T(n) = 1

时间复杂度

例如：T(n)=3n²+7n+6
用常数1代替运行时间中所有加法常数；T(n)=3n²+7n+1
修改后的运行次数函数中，只保留最高次项； T(n)=3n²
去除最高阶项的系数； T(n)=n² => O(n²)

常见的时间复杂度

随着问题规模n的不断增大，上述时间复杂度不断增大，算法的执行效率越低

常数阶O(1)

int i = 1;
int j = 1;
i++;
j++;

无论代码执行了多少行，只要是没有循环等复杂结构，那这个代码的时间复杂度就都是O(1)。
上述代码在执行的时候，它消耗的时间并不随着某个变量的增长而增长，那么无论这类代码有多长，即使有几万几十万行，都可以用O(1)来表示它的时间复杂度。

    i=i*2;

线性阶O(n)

int sum = 0
for(int i = 0;i<n;i++){
    int j=i;
    sum+=j;
}

for循环里面的代码会执行n次，它消耗的时间是随n的变化而变化的，因此时间复杂度为O(n)

线性对数阶

for(int j=0;j<n;j++){
    int i = 1;
	while(i<n){
    	i=i*2;
	}
}

将时间复杂度为的代码循环n次

平方阶

for(int i=0;i<n;i++){
    for(int j=0;j<n;j++){
        int m=0;
        m++;
    }
}

将时间复杂度为O(n)的代码循环n次

算法的空间复杂度

1、一个算法的空间复杂度(Space Complexity)定义为该算法所耗费的存储空间，它也是问题规模n的函数。

2、在做算法分析时，主要讨论的是时间复杂度。从用户使用体验上看，更看重的程序执行的速度。一些缓存产品(redis, memcache)和算法(基数排序)本质就是用空间换时间.
