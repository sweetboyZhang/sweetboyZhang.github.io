1、数据库的三范式是什么

第一范式：数据表必须是一个二维数据表，列不可再分，强调列的原子性，即表中的某一属性不能有几个值。

第二范式：必须建立在第一范式的基础之上，即满足第二范式必须满足第一范式。第二范式要求每一行必须被唯一标识，即表中必须要有一个主键，没有包含主键的列必须完全依赖主键。

第三范式：建立在第二范式的基础之上，要求每一个非主属性不能传递依赖候选键。
举例来说：Employee(emp_id,emp_name,emp_age,dept_id,dept_name,dept_info)，当员工表中emp_id能够唯一确定员工员工信息，但是dept_name可由dept_id唯一确定，此时，该表不符合第三范式，此时可以删除除了dept_id之外的其他部门信息，把所有部门信息单独建立一张部门表。

2、数据库的事务

什么是事务？

多条sql语句组成的一个执行单元，要么全部执行，要么都不执行。

事务的特性：原子性、一致性、隔离性、持久性

  • 原子性：组成事务的多个数据库操作是一个不可分割的原子单元，只有所有操作都成功，事务才会被提交，任何一个操作失败，已经执行的都必须撤销，让数据库恢复原来的状态。
  • 一致性：事务提交成功后，数据库所处的状态和它的业务规则是一致的，即数据不会被破坏。
  • 隔离性：不同事务拥有各自的数据空间，它们的操作不会对各自产生影响。
  • 持久性：一旦事务提交成功，事务中的所有操作必须持久化到数据库中。

3、数据库的索引

索引就是加快检索数据的方法。

mysql有四种不同的索引：主键索引(primary)、唯一索引(unique)、普通索引、全文索引

索引并非越多越好，创建索引页需要消耗资源，一是增加了数据库的存储空间，二是在插入和删除时需要花费较多的时间去维护。

4、drop、delete与truncate的区别

delete和truncate只能删除数据，不能删除表的结构

速度：drop>truncate>delete

truncate不可回滚

5、什么是视图

视图是一种虚拟的表，具有和物理表相同的功能。可以对视图进行增、删改、查操作，视图通常是一个表或多个表的行或列的子集。对视图的修改不影响基本表。

6、什么是内连接、左外连接、右外连接

内连接：用来匹配两张表中相关联的记录。

左外连接：除了匹配两张表中相关联的记录外，还会匹配左表剩余的记录，右表中未匹配的字段用NULL表示。

右外连接：除了匹配两张表中相关联的记录外，还会匹配右表剩余的记录，左表中未匹配的字段用NULL表示。

7、并发事务带来哪些问题？

脏读：对于两个事务t1,t2，t1读取了已经被t2更新但还没提交的数据，若t2回滚，t1读取的数据就是临时且失效的。

不可重复读：对于两个事务t1,t2，t1读取了一个字段，然后t2更新了该字段。之后，t1再次读取同一个字段，值就不同了。

幻读：对于两个事务t1、t2，t1从一个表中读取了一个字段，然后t2在该表中插入了一些新的行。之后，若t1再次读取同一个表，就会多出几行。

9、mysql数据类型

1、整数类型
Tinyint：1个字节，有符号：-128到127 ，无符号(unsigned)：0到255
Smallint：2个字节，有符号：-32768到32767，无符号数：0到65535
Mediumint：3个字节
int/Integer：4个字节
Bigint：8个字节

2、实数类型
float、double、decimal(效率高，精度高)

3、字符串类型
varchar(变长字符串)、char(定长字符串)、text、blob

4、日期和时间类型
timestamp(效率高)、datetime

10、创建索引的三种方式

方式一：创建表的时候

CREATE TABLE user(
	id INT atuo_increment PRIMARY KEY,
    name varchar(20),
    age TINYINT
)

方式二：使用alter table命令，创建普通索引、UNIQUE索引或PRIMARY KEY索引。

ALTER TABLE table_name ADD INDEX index_name (column_list);

方式三：使用create index命令，增加普通索引或unique索引

CREATE INDEX index_name ON table_name (column_list);
